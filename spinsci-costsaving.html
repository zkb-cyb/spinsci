Target AI Containment Rate 65%

Target % of calls fully resolved by AI — no live agent transfer needed.

Target must exceed Current AI Containment Rate.
▶ How is this calculated?
Containment Savings = additional calls AI handles × cost avoided per call Monthly Savings = (Target Containment% − Current Containment%) × Monthly Calls × Cost/Agent Call
  • Target Containment% — the slider value you set above
  • Current Containment% — your baseline from the Assumptions panel
  • Monthly Calls — total inbound calls per month
  • Cost/Agent Call — fully-loaded cost when a live agent handles a call

Example: Moving from 15% → 65% containment on 500,000 calls at $3.50/call saves (0.65−0.15) × 500,000 × $3.50 = $875,000/month.

Target Repeat Call Rate 8%

Target % of calls that are follow-up repeats — lower is better. Driven by first-contact resolution (FCR).

Target must be lower than Current Repeat Rate.
▶ How is this calculated?
Repeat Call Savings = fewer repeat calls × cost avoided per call Monthly Savings = (Current Repeat Rate% − Target Repeat Rate%) × Monthly Calls × Cost/Agent Call
  • Current Repeat Rate% — baseline from Assumptions panel
  • Target Repeat Rate% — the slider value you set above
  • Monthly Calls — total inbound calls per month
  • Cost/Agent Call — cost per live-agent-handled call

Example: Reducing repeat rate from 18% → 8% on 500,000 calls at $3.50/call saves (0.18−0.08) × 500,000 × $3.50 = $175,000/month.

Target EHR Automation Success Rate 98%

Target % of EHR write-backs completing without manual correction. Higher rate = fewer rework events.

Target must exceed Current EHR Success Rate.
▶ How is this calculated?
Rework Savings = fraction of current rework events eliminated × cost per event Fraction Eliminated = (Target EHR% − Current EHR%) ÷ (100% − Current EHR%) Monthly Savings = Fraction Eliminated × Monthly Rework Events × Cost per Rework Event
  • Target EHR% — slider value above
  • Current EHR% — baseline from Assumptions panel
  • Monthly Rework Events — current number of failed EHR transactions per month
  • Cost per Rework Event — staff + system cost to manually fix one failed transaction

Why this formula? The denominator (100% − Current EHR%) normalizes against the improvable gap. If you are already at 92% success, the remaining 8% failure rate represents all current rework events. Moving from 92% → 98% eliminates (6/8) = 75% of those events.

Example: 75% of 8,000 events at $8.00 each = $48,000/month.

📐 Formula Reference

All three value drivers use the same principle: delta improvement × unit volume × unit cost. Below is the complete formula set.

1. Call Containment Efficiency

Monthly = (TargetContainment − CurrentContainment) × MonthlyCalls × CostPerAgentCall Annual = Monthly × 12

Interpretation: Each percentage point of additional containment eliminates that share of calls from live-agent queues. The saving is the agent cost avoided for those calls.

2. Repeat Call Reduction

Monthly = (CurrentRepeatRate − TargetRepeatRate) × MonthlyCalls × CostPerAgentCall Annual = Monthly × 12

Interpretation: Each repeat call is effectively a second paid handling of the same patient issue. First-contact resolution (FCR) driven by SpinSci's EHR write-backs prevents that second call entirely.

3. EHR / Rework Mitigation

FractionEliminated = (TargetEHR − CurrentEHR) ÷ (1 − CurrentEHR) Monthly = FractionEliminated × MonthlyReworkEvents × CostPerReworkEvent Annual = Monthly × 12

Why normalize by (1 − CurrentEHR)? The monthly rework events represent the current failure population. The denominator converts the raw EHR improvement into the fraction of that failure population that is eliminated — giving an accurate rework count reduction regardless of starting baseline.

Example: Current EHR 92% → Target 98%, 8,000 rework events, $8.00/event:
Fraction = (0.98−0.92)/(1−0.92) = 0.75 → 6,000 events eliminated → $48,000/month saved.

4. Annual & 3-Year Totals

Annual Savings = (Containment + Repeat + Rework) × 12 3-Year Benefit = Annual Savings × 3

Note: The 3-year figure assumes steady-state performance and flat volume. For growth-adjusted projections, apply your expected call volume CAGR to Monthly Calls before using this calculator.
